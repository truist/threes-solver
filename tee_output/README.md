# tee_output

A small create that provides "tee" functionality for the current process.
It works by redirecting stdout and stderr to pipes, the reading those pipes form a background thread, writing both to their original targets and to a specified file.

This seemed like something that would have already existed, but I couldn't find it, so I made it.

One way you might use it:

```rust
use tee_output::{init_log_file, TeeGuard};

fn init_logging(file_name: &str) -> TeeGuard {
    match init_log_file(file_name) {
        Ok(guard) => guard,
        Err(e) => panic!("Error: Could not open log file {}: {}", file_name, e),
    }
}
```

Then call `init_logging` (or whatever wrapper function you want to use, if any) and hold on to the returned `TeeGuard` for as long as you need the "tee" behavior to continue.
As soon as the `TeeGuard` is dropped, the tee will stop.
